# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in prodlint, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please email **marcovecchioanthony@gmail.com** with:

- A description of the vulnerability
- Steps to reproduce
- Potential impact

We aim to respond within 48 hours and will work with you to understand and address the issue before any public disclosure.

## Scope

This policy covers the `prodlint` npm package, the GitHub Action (`action.yml`), and the MCP server (`prodlint-mcp`).

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.5.x   | Yes       |
| < 0.5   | No        |
